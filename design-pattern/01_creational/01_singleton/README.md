# 单例模式 (Singleton Pattern)

## 特点
- 保证一个类仅有一个实例，并提供一个访问它的全局访问点
- 私有化构造函数，防止外部直接创建实例
- 提供一个全局访问点来获取唯一实例

## 实现特点
1. 懒汉式：第一次使用时才初始化
2. 饿汉式：程序启动时就初始化
3. 双重检查锁定：线程安全且性能优化的实现方式

## 常见使用场景
1. 数据库连接池
2. 配置管理器
3. 日志记录器
4. 缓存管理器

## 优点
1. 保证唯一实例，节省内存
2. 避免频繁创建销毁实例
3. 提供了对唯一实例的受控访问

## 缺点
1. 单例模式可能隐藏组件之间的依赖关系
2. 单例模式违反单一职责原则
3. 在并发环境下需要特别注意线程安全
